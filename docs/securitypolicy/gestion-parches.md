# Gestion de Parches y Actualizaciones

## 1. Objetivo
Mantener sistemas y aplicaciones actualizados para reducir vulnerabilidades.

## 2. Alcance
Servicios subcontratados, aplicaciones propias, endpoints y herramientas SaaS.

## 3. Politica
- Actualizaciones automaticas cuando el proveedor lo permita.
- Revision trimestral de versiones criticas.
- Correccion prioritaria de vulnerabilidades altas/criticas.

## 4. Tiempos objetivo (minimos)
- Critico: 30 dias.
- Alto: 60 dias.
- Medio: 90 dias.

## 5. Evidencia
Registro trimestral de revisiones y cambios aplicados.
